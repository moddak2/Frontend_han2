Fronte 0.1.0 사용설명서

1. 프로젝트 개요
이 프로젝트는 설계도와 보고서를 바탕으로 만든 프론트엔드 데스크톱 앱의 기본 틀입니다.
처음부터 백엔드 데이터를 바로 연결하는 방식이 아니라, 먼저 화면 구조와 디자인 시스템, 데이터가 들어갈 자리를 확실히 잡아둔 상태입니다.
현재는 하드코딩된 샘플 데이터로 동작하며, 나중에 백엔드 API가 연결되면 데이터만 교체해서 그대로 사용할 수 있도록 구성했습니다.
Electron으로 감싸서 더블클릭으로 실행 가능한 데스크톱 앱 형태도 함께 준비했습니다.
보안채팅 활성화와 사용자의 명시적인 동의를 기반으로 화면 또는 특정 창을 공유하고, 앱 내부에서 실시간으로 미리 볼 수 있습니다.

2. 프로젝트 목표
- 앱처럼 보이는 실제 UI를 만든다.
- 설계도와 보고서의 흐름을 반영한 화면 구성을 만든다.
- 백엔드 데이터가 들어올 수 있는 데이터 슬롯과 계약 초안을 만든다.
- 하드코딩 데이터로 먼저 실행 가능한 상태를 만든다.
- 나중에 백엔드 연동 시 구조를 크게 바꾸지 않도록 한다.
- 사용자가 직접 허용한 화면만 안전하게 공유하고 즉시 중지할 수 있도록 한다.

3. 현재 구성 방식
현재 화면은 크게 다섯 개의 영역으로 나뉩니다.

- 좌측 네비게이션
  - Overview
  - Reports
  - Data Map
  - Activity
  - Settings
- 상단 헤더
  - 앱 제목
  - 간단한 설명
  - Sync 상태 버튼
- 메인 콘텐츠 영역
  - 개요 화면
  - 보고서 목록과 상세
  - 데이터 연동 자리
  - 활동 로그
  - 설정 정보
- 하단 상태 영역
  - 보안채팅 활성화 상태
  - 채팅 모드
  - 현재 선택된 화면
  - 필터 상태
  - 선택된 보고서

4. 만든 기능 설명
4-1. Overview 화면
- 앱 전체 구조를 보여주는 시작 화면입니다.
- 디자인 시스템과 데이터 shell 개념을 보여줍니다.
- 핵심 지표 카드가 보입니다.
- 작업 흐름 카드가 보입니다.
- 보안채팅을 켜거나 끌 수 있습니다.
- 공유할 화면 또는 창을 선택하고 실시간 미리보기를 확인할 수 있습니다.
- 화면 공유가 활성화되면 공유 대상과 캡처 상태를 표시합니다.
- 공유 중지 버튼으로 캡처를 즉시 종료할 수 있습니다.

4-2. Reports 화면
- 보고서 목록을 볼 수 있습니다.
- 검색 기능이 있습니다.
- 상태 필터 기능이 있습니다.
- 목록에서 항목을 클릭하면 오른쪽 상세 패널 내용이 바뀝니다.
- 보고서의 카테고리, 담당자, 상태, 진행률, 태그를 확인할 수 있습니다.

4-3. Data Map 화면
- 백엔드가 나중에 연결될 엔드포인트 자리와 데이터 계약을 보여줍니다.
- /api/profile
- /api/dashboard/summary
- /api/reports
- /api/activity
- /api/reports/:id
- 실제 API가 연결되면 이 자리의 샘플 데이터를 교체하는 방식으로 확장할 수 있습니다.

4-4. Activity 화면
- 최근 작업 로그를 시간순으로 보여줍니다.
- 승인, 연동 준비, UI 검토, 데이터 갱신 같은 이벤트를 확인할 수 있습니다.

4-5. Settings 화면
- 화이트 모드와 다크 모드를 선택할 수 있습니다.
- 글자 크기를 작게, 기본, 크게 중에서 선택할 수 있습니다.
- 선택한 테마와 글자 크기는 다음 실행에도 유지됩니다.
- mock data mode, responsive layout, 상태 변경 알림 같은 설정을 제공합니다.

4-6. 보안채팅과 화면 공유
- 화면 공유는 보안채팅이 활성화된 경우에만 시작할 수 있습니다.
- 사용자가 화면 선택 및 공유 시작 버튼을 눌러야 화면 목록이 표시됩니다.
- 전체 화면이나 특정 앱 창 중 하나를 직접 선택할 수 있습니다.
- 선택한 화면은 앱 내부 미리보기에 실시간으로 표시됩니다.
- 현재 단계에서는 캡처 화면을 외부 서버로 전송하거나 AI로 분석하지 않습니다.
- 공유 중에는 활성 상태가 표시되며 언제든 즉시 중지할 수 있습니다.
- 화면 목록 조회는 Electron의 desktopCapturer와 안전한 preload IPC 브리지를 사용합니다.

5. 디자인 구성
디자인은 화이트 테마를 기본으로 하며 다크 테마도 선택할 수 있는 대시보드 스타일입니다.

- 배경은 그라데이션과 빛 번짐 효과를 사용했습니다.
- 카드들은 반투명 글래스 느낌으로 처리했습니다.
- 정보 우선순위를 위해 상단 요약, 중간 작업, 하단 계약 정보로 나눴습니다.
- 데스크톱, 태블릿, 모바일에서 반응형으로 작동하도록 만들었습니다.

6. 데이터 구성 방식
현재 데이터는 모두 하드코딩된 샘플입니다.

사용된 주요 데이터 예시:
- metrics: 핵심 지표 카드
- workflow: 화면 흐름과 작업 상태
- reports: 보고서 목록
- activities: 활동 로그
- endpoints: 백엔드 API 자리
- apiContract: 백엔드 연결용 데이터 구조 예시

즉, 지금은 UI 검증과 설계 검토용 샘플 데이터로 동작하고, 이후에는 이 구조를 유지한 채 API 응답으로 교체하면 됩니다.

7. 데스크톱 앱 실행 방식
이 프로젝트는 Electron 메인 프로세스를 통해 데스크톱 창으로 실행할 수 있습니다.

개발용 실행:

run-desktop.cmd

Windows에서는 프로젝트 폴더의 run-desktop.cmd를 실행하는 방식을 권장합니다.
이 스크립트는 PowerShell의 npm.ps1 실행 정책 문제와 ELECTRON_RUN_AS_NODE 환경 변수 충돌을 피하고, Vite 렌더러와 Electron 창을 함께 띄웁니다.

터미널에서 직접 실행하려면:

npm.cmd run desktop

개발 서버는 5173 포트를 고정해서 사용합니다. 포트가 이미 사용 중이면 다른 포트로 자동 변경하지 않고 오류를 표시해 Electron과 렌더러 주소가 어긋나는 문제를 방지합니다.

Windows 실행 파일 생성:

npm run dist:win

이 명령은 GitHub Release나 로컬 산출물로 사용할 수 있는 실행 파일을 만드는 용도입니다.

8. 실행 방법
터미널에서 아래 순서로 실행합니다.

npm install
npm.cmd run dev

브라우저에서 접속 주소:
http://localhost:5173/

9. 빌드 확인 방법
배포용 빌드가 정상인지 확인하려면 아래 명령어를 사용합니다.

npm run build

12. 현재 상태 요약
Fronte 0.1.0은 단순한 정적 페이지가 아니라 Electron 데스크톱 앱으로 실행되는 프론트엔드 기반입니다.
현재는 디자인과 화면 구조, 보안채팅 동의, 화면 선택, 실시간 로컬 미리보기, 안정적인 Windows 실행 환경까지 구현되어 있습니다.
화면 데이터는 아직 외부로 전송하거나 AI로 분석하지 않으며, 다음 단계에서 로컬 필터링과 명시적인 추가 동의를 전제로 분석 기능을 연결할 수 있습니다.

13. API 및 저장소 연동 기반
- /api/v1/health, /analyze, /model/info, /ruleset 계약이 구현되어 있습니다.
- 상단 Sync data 버튼으로 엣지 서버 상태를 다시 확인할 수 있습니다.
- 서버 미응답 시 입력 기능을 중단하지 않고 로컬 기본 검사로 전환합니다.
- Data Map 화면에는 실제 보안 키보드 API 경로와 위험 분석 응답 구조를 표시합니다.
- VITE_USE_MOCKS=true이면 서버 없이 mock 응답을 사용하고 false이면 실제 엣지 서버를 사용합니다.
- 입력 원문은 저장하지 않으며 위험 등급, 유형, 사용자 조치만 저장합니다.
- 웹과 Electron은 localStorage 어댑터를 사용하고 Android에서는 같은 계약의 SQLite 구현체로 교체합니다.
