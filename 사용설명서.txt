Frontend_han2 사용설명서

1. 프로젝트 개요
이 프로젝트는 설계도와 보고서를 바탕으로 만든 프론트엔드 데스크톱 앱의 기본 틀입니다.
처음부터 백엔드 데이터를 바로 연결하는 방식이 아니라, 먼저 화면 구조와 디자인 시스템, 데이터가 들어갈 자리를 확실히 잡아둔 상태입니다.
현재는 하드코딩된 샘플 데이터로 동작하며, 나중에 백엔드 API가 연결되면 데이터만 교체해서 그대로 사용할 수 있도록 구성했습니다.
Electron으로 감싸서 더블클릭으로 실행 가능한 데스크톱 앱 형태도 함께 준비했습니다.

2. 프로젝트 목표
- 앱처럼 보이는 실제 UI를 만든다.
- 설계도와 보고서의 흐름을 반영한 화면 구성을 만든다.
- 백엔드 데이터가 들어올 수 있는 데이터 슬롯과 계약 초안을 만든다.
- 하드코딩 데이터로 먼저 실행 가능한 상태를 만든다.
- 나중에 백엔드 연동 시 구조를 크게 바꾸지 않도록 한다.

3. 현재 구성 방식
현재 화면은 크게 다섯 개의 영역으로 나뉩니다.

- 좌측 네비게이션
  - Overview
  - Reports
  - Data Map
  - Activity
  - Settings
- 상단 헤더
  - 앱 제목
  - 간단한 설명
  - Sync 상태 버튼
- 메인 콘텐츠 영역
  - 개요 화면
  - 보고서 목록과 상세
  - 데이터 연동 자리
  - 활동 로그
  - 설정 정보
- 하단 상태 영역
  - 현재 선택된 화면
  - 검색어
  - 필터 상태
  - 선택된 보고서

4. 만든 기능 설명
4-1. Overview 화면
- 앱 전체 구조를 보여주는 시작 화면입니다.
- 디자인 시스템과 데이터 shell 개념을 보여줍니다.
- 핵심 지표 카드가 보입니다.
- 작업 흐름 카드가 보입니다.

4-2. Reports 화면
- 보고서 목록을 볼 수 있습니다.
- 검색 기능이 있습니다.
- 상태 필터 기능이 있습니다.
- 목록에서 항목을 클릭하면 오른쪽 상세 패널 내용이 바뀝니다.
- 보고서의 카테고리, 담당자, 상태, 진행률, 태그를 확인할 수 있습니다.

4-3. Data Map 화면
- 백엔드가 나중에 연결될 엔드포인트 자리와 데이터 계약을 보여줍니다.
- /api/profile
- /api/dashboard/summary
- /api/reports
- /api/activity
- /api/reports/:id
- 실제 API가 연결되면 이 자리의 샘플 데이터를 교체하는 방식으로 확장할 수 있습니다.

4-4. Activity 화면
- 최근 작업 로그를 시간순으로 보여줍니다.
- 승인, 연동 준비, UI 검토, 데이터 갱신 같은 이벤트를 확인할 수 있습니다.

4-5. Settings 화면
- 현재 앱이 어떤 방식으로 동작하는지 안내합니다.
- mock data mode, responsive layout, backend contract 같은 설명을 보여줍니다.

5. 디자인 구성
디자인은 어두운 테마 기반의 대시보드 스타일입니다.

- 배경은 그라데이션과 빛 번짐 효과를 사용했습니다.
- 카드들은 반투명 글래스 느낌으로 처리했습니다.
- 정보 우선순위를 위해 상단 요약, 중간 작업, 하단 계약 정보로 나눴습니다.
- 데스크톱, 태블릿, 모바일에서 반응형으로 작동하도록 만들었습니다.

6. 데이터 구성 방식
현재 데이터는 모두 하드코딩된 샘플입니다.

사용된 주요 데이터 예시:
- metrics: 핵심 지표 카드
- workflow: 화면 흐름과 작업 상태
- reports: 보고서 목록
- activities: 활동 로그
- endpoints: 백엔드 API 자리
- apiContract: 백엔드 연결용 데이터 구조 예시

즉, 지금은 UI 검증과 설계 검토용 샘플 데이터로 동작하고, 이후에는 이 구조를 유지한 채 API 응답으로 교체하면 됩니다.

7. 데스크톱 앱 실행 방식
이 프로젝트는 Electron 메인 프로세스를 통해 데스크톱 창으로 실행할 수 있습니다.

개발용 실행:

npm run desktop

이 명령은 Vite 렌더러와 Electron 창을 함께 띄웁니다.

Windows 실행 파일 생성:

npm run dist:win

이 명령은 GitHub Release나 로컬 산출물로 사용할 수 있는 실행 파일을 만드는 용도입니다.

8. 실행 방법
터미널에서 아래 순서로 실행합니다.

npm install
npm run dev

브라우저에서 접속 주소:
http://localhost:5173/

9. 빌드 확인 방법
배포용 빌드가 정상인지 확인하려면 아래 명령어를 사용합니다.

npm run build

10. 파일 구조 요약
- package.json: 프로젝트 설정과 실행 스크립트
- index.html: 앱 진입점
- src/App.tsx: 메인 화면과 기능 로직
- src/styles.css: 전체 스타일
- electron/main.cjs: 데스크톱 앱 메인 프로세스
- electron/preload.cjs: 전역 브리지 설정
- README.md: 간단한 프로젝트 안내
- 사용설명서.txt: 이 문서

11. 앞으로 확장할 수 있는 방향
- 로그인 화면 추가
- 실제 백엔드 API 연결
- 보고서 상세 페이지 분리
- 차트 및 통계 시각화 추가
- 사용자 권한별 UI 분기
- 폼 입력과 저장 기능 추가
- 알림, 검색 고도화, 테이블 기능 추가

12. 현재 상태 요약
이 프로젝트는 단순한 정적 페이지가 아니라, 실제 앱처럼 보이는 구조를 가진 프론트엔드 기반입니다.
지금 단계에서는 디자인과 화면 구조를 먼저 고정했고, 다음 단계에서 백엔드 데이터를 연결하면 바로 확장할 수 있는 상태입니다.

13. 백엔드 연동 구조
- API 명세서 v2.0의 /api/v1/health, /analyze, /model/info, /ruleset을 지원합니다.
- 상단 서버 상태는 실제 health API 또는 mock 결과를 표시합니다.
- 서버 미응답 시 기본 검사 모드(degraded mode)로 전환합니다.
- .env.example을 .env.local로 복사하여 서버 주소와 API Key를 설정합니다.
- VITE_USE_MOCKS=true는 샘플 응답, false는 실제 엣지 서버를 사용합니다.

14. 로컬 데이터 저장 원칙
- 웹 프로토타입은 localStorage를 사용합니다.
- Android 앱에서는 SecurityRepository 계약을 SQLite 구현으로 교체합니다.
- 위험 로그, 설정, 보호자 정보, 차단 이력만 저장하며 입력 원문은 저장하지 않습니다.
